## Architecture Notes

- **Data flow:** SQLite is the local-first store; Supabase is the backend for auth and cloud sync; Google Drive is used for full backups.
- **Auth:** Google OAuth via Supabase. Access token stored in AsyncStorage (short-lived), refresh token in SecureStore (encrypted). A Supabase Edge Function handles server-side token refresh so the client secret is never in the app.
- **Routing:** Expo Router file-based. Auth state is checked in `app/_layout.jsx` and redirects to `(auth)` or `(tabs)` accordingly.
- **Ads:** Banner ads on dashboard, interstitial shown at most once per day on budget screen. Lazy-imported to avoid startup delay.
- **Offline:** `offlineQueue.js` queues mutations when there is no network (netinfo).
- **Metro config** excludes `supabase/functions/**` from bundling — those are server-side Deno functions.

| Architectures | armeabi-v7a, arm64-v8a, x86, x86_64 |

**Important:** Do NOT hardcode `ndkVersion` inside `android/app/build.gradle`. It must come from `rootProject.ext.ndkVersion` (defined in `android/build.gradle`). Previously this was set to NDK 26 there, which broke the C++ codegen.

## CI/CD (GitHub Actions)

Workflow: `.github/workflows/build.yml` — triggers on push to `main`.

Key steps in order:
1. `npm install --legacy-peer-deps`
2. Setup Java 17 (Temurin)
3. Setup Android SDK (`android-actions/setup-android@v3`)
4. **`sdkmanager "ndk;27.0.12077973" "cmake;3.31.1"`** — must be explicit; the runner only has cmake 3.22.1 pre-installed
5. Cache Gradle (`~/.gradle/caches`, `~/.gradle/wrapper`)
6. `npx expo prebuild --platform android` (regenerates the android/ directory)
7. `npx expo export --platform android` → copies `.hbc` bundle to `android/app/src/main/assets/`
8. Decode keystore from `KEYSTORE_BASE64` secret
9. `./gradlew assembleRelease` → APK artifact (1-day retention)
10. `./gradlew bundleRelease` → AAB artifact (1-day retention)

Secrets required: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `KEYSTORE_BASE64`, `KEYSTORE_PASSWORD`, `KEY_ALIAS`, `KEY_PASSWORD`.

## Patches & Known Quirks

### react-native+0.81.5.patch
`patches/react-native+0.81.5.patch` — replaces `std::format("{}%", dimension.value)` with `std::to_string(dimension.value) + "%"` in `ReactCommon/react/renderer/core/graphicsConversions.h`. Applied automatically via `patch-package` on `npm install` (postinstall hook).

### Pinned / excluded packages
- `expo-font` is in `expo.install.exclude` — prevents Expo from auto-downgrading it.
- `react` and `react-dom` are overridden to 19.1.0 in `overrides` to prevent peer deps from pulling in older versions.
- `--legacy-peer-deps` is required everywhere due to peer dep conflicts across the Expo/Firebase/RN ecosystem.

### Lint relaxed
```gradle
lint { checkReleaseBuilds false; abortOnError false }
```
Necessary because Expo/Firebase deps generate lint warnings that would otherwise abort the release build.

### async-storage version
Must be **2.x** (currently 2.1.2). Version 1.23.x does not have a correct Fabric-compatible spec for RN 0.81's New Architecture codegen and causes `ShadowNodes.cpp` to fail to compile.
